## ✨ Features 

- ✅ **Zero authentication** — because identity is inherently an illusion  
- ✅ **Zero database** — chat.txt is the truth  
- ✅ **Zero frontend framework** — pure HTML, as God originally intended  
- ✅ **Zero POST requests** — GET is enough, and you deserve only GET  
- ✅ **Zero configuration** — want to change settings? Go edit the server source code!  
- ✅ **Human-readable backup** — right-click → open with txt  
- ✅ **Built-in admin panel** — it's called the file editor 

---


## 🚀 Quick Start 

1. Clone the repository git clone https://github.com/XieXianGuang/txtchat.git


2. cd txtchat


3. Run ./txtchat 

4. Start shouting  
Open your browser and enter the following in the address bar: `http://localhost:8080` 

---


## 📡 Usage Instructions 

### Send a message  
Just like in 1996, type the message into the browser's address bar:  
http://localhost:8080/send?msg=Hello&user=txtchat  

### View messages  
Simply visit `/` and manually refresh. Patience is a virtue. 

---


## 💾 Backup  
Right-click `chat.txt` → Download  
Congratulations, you've mastered disaster recovery technology. 

---


## 🔧 Admin Panel  
Want to delete messages? Edit history? Pin announcements?  
Open `chat.txt`  
This process is formally known as **"governance."**

#### 🧠 Design Philosophy  
Modern chat systems are built by teams.  
This one was written with frustration. 

We believe:  
- Databases are for cowards  
- POST is a form of bureaucracy  
- CSS is a moral hazard  
- Configuration files are unnecessary complexity  
- Anything that can't be fixed with text shouldn't exist 

---


## 🛡️ Security Notes  
- No login → No password leakage  
- No database → No SQL injection  
- No JavaScript → No XSS  
- Only GET requests → Browser history serves as audit log
